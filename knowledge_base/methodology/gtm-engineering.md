# GTM Engineering

GTM Engineering (GTME) is the discipline of applying software engineering practices to Go-to-Market execution. Instead of manual prospecting, list building, and email composition, a GTM engineer writes code: enrichment pipelines, scoring algorithms, and automated outbound sequences.

theHAWG-BIGDAWG is a GTM engineering experiment — a 90-day case study proving that engineering-led GTM produces better pipeline than traditional SDR workflows. The experiment uses [[thehog-ai-company]] as the data layer, [[icp-scoring-engine]] as the analytical layer, and [[outbound-playbook]] as the execution layer.

## Key Points

- GTM engineering treats pipeline as a system, not a manual process
- Three layers: **Strategic** ([[thirty-sixty-ninety-plan]]), **Analytical** ([[icp-scoring-engine]] + [[icp-segmentation]]), **Execution** ([[outbound-playbook]] + enrichment)
- The founding GTM engineer workflow follows [[sense-orient-act-deposit]] — same loop as Context OS
- Key metric: qualified conversations per engineering hour vs. per SDR hour
- Tools: Hog API for data, MCP for agent access, scoring logic for prioritization
