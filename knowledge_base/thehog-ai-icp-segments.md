# TheHog.ai ICP Segments

## ICP 1: Solo founders and lean startups

- Profile: seed to Series A teams with minimal GTM headcount.
- Pain: manual research, no dedicated growth team, fragmented tools.
- Signal: founder-led sales posts, hiring first GTM role, public discussion about growth without headcount.
- Value: command center that compresses research-to-execution.

## ICP 2: AI agent developers and AI infrastructure teams

- Profile: teams building AI agents that need real-time company, people, social, sentiment, news, and web context.
- Pain: stitching multiple APIs into an agent data layer.
- Signal: developer forum activity, GitHub agent work, data pipeline or AI engineer hiring.
- Value: one structured intelligence API for agent context.

## ICP 3: GTM engineers and RevOps teams at B2B SaaS

- Profile: Series A to B teams running Clay, Apollo, Smartlead, HubSpot, n8n, or similar stacks.
- Pain: static data, enrichment waterfall complexity, weak real-time signal layer.
- Signal: Clay workflow posts, GTM engineer hiring, Apollo plus automation stack usage.
- Value: unified company-first search, people discovery, enrichment, and live monitoring.
