# ICP: GTM Engineer

GTM engineers are technical practitioners who write code to drive pipeline — they build enrichment pipelines, scoring engines, and automated outbound systems. They evaluate tools by API quality, data coverage, and integration flexibility. This is the persona that the [[thehog-ai-api]] and [[mcp-integration]] are built for.

This segment has medium urgency (they already have a pipeline, they're optimizing it) and medium-to-high budget authority (they influence or control tool spend). The value prop is data quality and API ergonomics: how reliable is the enrichment data, and how clean is the integration.

## Key Points

- **Pain:** Existing enrichment tools have poor APIs, rate limits, stale data, or require UI interaction
- **Decision criteria:** API documentation quality, data freshness, batch support, MCP compatibility
- **Buying signal:** Currently using Clay, Apollo, Clearbit, or building custom scrapers
- **Disqualifier:** Non-technical marketer who won't touch an API
- **Messaging angle:** "Enrichment API that a GTM engineer actually wants to use — batch, async, metered, MCP-native"
