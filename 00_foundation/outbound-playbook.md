# Outbound Playbook

## Principle

Outbound should be signal-led, account-first, and grounded in TheHog.ai data. Use the graph to decide who to contact, why now, and what proof to bring.

## Workflow

1. Select accounts from company search or monitor events.
2. Map each account to an ICP node.
3. Find relevant people with people search.
4. Enrich only contacts that pass account and persona fit.
5. Draft outreach from account-specific signals.
6. Log outcomes back to `findings.md` and the relevant account/persona node.

## Campaign tracks

### AI developer / API audience

- Trigger: agent data layer pain, enrichment API discussions, data pipeline hiring, public AI agent work.
- Message: one structured API call instead of stitched data providers.
- Proof: schema-conformant deep research and unified company/person/social context.

### Lean founder / solo GTM

- Trigger: founder-led growth, first GTM hire, growth without headcount, manual research complaints.
- Message: command center compresses research-to-execution.
- Proof: first account list, contact list, and live signal monitor in one workflow.

### GTM engineer / RevOps

- Trigger: Clay workflows, Apollo enrichment, CRM data quality issues, GTM engineer hiring.
- Message: real-time signal layer plus company-first enrichment.
- Proof: fewer API calls and cleaner account context than a multi-provider waterfall.
